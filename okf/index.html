Overview

A REST backend (Java 21 / Spring Boot, JWT auth) that lets a group log bowling sessions, enter frame-by-frame scores per bowler per game, and compute win/loss points and a session leaderboard.

Entities

  • User — login account
  • Role — authorization role (ADMIN, USER)
  • Bowler — a player, optionally linked to a User
  • BowlingSession — a date/location grouping of games
  • Game — one game within a session
  • BowlerGame — a bowler's participation/result in a game
  • Frame — one of the 10 frames within a BowlerGame

API

  • Auth — /api/auth/* login, register, refresh, current user
  • Bowlers — /api/bowlers/* CRUD
  • Sessions — /api/sessions/* sessions and their games
  • Scores — /api/scores/* frame entry and leaderboard
