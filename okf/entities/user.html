Relationships

  • Many-to-many with Role via join table user_roles.
  • Optionally one-to-one with Bowler (Bowler.user).

Gotcha: roles must be a mutable Set

When constructing a User (e.g. in UserService.registerUser, DataInitializer), always assign roles a mutable collection (new HashSet<>(...)), never Set.of(...). Hibernate wraps whatever collection instance is passed at first save as the managed PersistentSet; an immutable Set.of(...) there causes UnsupportedOperationException the next time that user entity is save()'d (e.g. in UserService.changePassword), because merge tries to clear() it.
